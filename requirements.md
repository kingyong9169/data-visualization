# 요구사항 분석

## 제약 조건

1. React
2. Typescript

## 기능적 요구사항

1. 데이터 요청

- []: Endpoint 기준 10개 이상의 Open API를 사용하여 대시보드 구성
- []: 실시간 데이터 요청(5초 주기로 데이터 갱신)
- []: 본인의 분석관점을 반영하여 Open API를 선택하고 대시보드 구성 시 긍정 평가 요소

2. 컴포넌트

- []: 1개 이상의 인포매틱스 위젯(지표 이름과 값을 단순 텍스트 형식으로 표시)
- []: 1개 이상의 막대 차트 위젯
- []: 1개 이상의 라인 차트 위젯(네트워크 요청 부하, 데이터 상태 관리 측면에서 성능 최적화 방법 고민)
- []: 반응형 레이아웃
- []: 차트 종류 변경, 차트 확대, 도움말 표시, 임계선 표시 등 사용자 편의 기능은 자유롭게 추가
